API Doc

Overview

Marvel Characters API is a CRUD RESTful web API for managing data related to Marvel comic characters.

Standards and Conventions

All string inputs should be lowercase except for the origin description, which may include uppercase characters. Nicknames should be surrounded by single quotes. Earth universes should be formatted like earth-616.

Endpoints

There is one unencrypted production endpoint with the value http://127.0.0.1:5000/characters.

Authorization

In your profile you will find your API key, which you will need to provide when making HTTP requests.

To authorize access to the API, add a custom header named x-access-token and your API key as its value.

Requests

You may CREATE CHARACTER, GET SINGLE CHARACTER, GET ALL CHARACTERS, UPDATE CHARACTER, or DELETE CHARACTER.

Variables

Data for each character is contained in the following variables, one of which, current_alias, is not nullable.

  • comics_appeared_in -- the number of comics a character has appeared in
  • current_alias -- the current alias of a character, e.g., captain marvel
  • origin -- the background and origin story of the character
  • power -- one of the main powers of the character
  • real_name -- the real name of the character, e.g. peter parker
  • universe -- the primary fictional universe in which the character exists
Input data format

The body of CREATE CHARACTER and UPDATE CHARACTER should be in JSON data format like the following example: 


        {
            "comics_appeared_in": 819,
            "current_alias": "captain marvel",
            "origin": "Human/Kree Hybrid further altered...",
            "power": "energy manipulation",
            "real_name": "carol susan jane danvers ",
            "universe": "earth-616"
        }
Output data format

All outputs are in JSON data format.

CREATE CHARACTER

HTTP request: POST http://127.0.0.1:5000/characters

sample output:


        {
            "comics_appeared_in": 819,
            "current_alias": "captain marvel",
            "id": 11,
            "origin": "Human/Kree Hybrid further altered...",
            "power": "energy manipulation",
            "real_name": "carol susan jane danvers ",
            "universe": "earth-616"
        }
GET SINGLE CHARACTER

HTTP request: GET http://127.0.0.1:5000/characters/{id}

URL parameter {id} denotes which character to get, where {id} is an integer

sample output:


        {
            "comics_appeared_in": 10563,
            "current_alias": "captain america",
            "id": 2,
            "origin": "Human enhanced to the peak of human perfection...",
            "power": "super-soldier",
            "real_name": "steven 'steve' rogers",
            "universe": "earth-616"
        }

sample output:

            {

                "origin": "Human/Kree Hybrid further altered on...",

        ]
UPDATE CHARACTER

HTTP request: POST http://127.0.0.1:5000/characters/{id}

URL parameter {id} denotes which character to update, where {id} is an integer

sample output:


        {
            "comics_appeared_in": 15435,
            "current_alias": "spider-man",
            "id": 8,
            "origin": "Radioactive Spider bite during a science...",
            "power": "spider physiology",
            "real_name": "peter benjamin parker",
            "universe": "earth-616"
        }
DELETE CHARACTER

HTTP request: DEL http://127.0.0.1:5000/characters/{id}

URL parameter {id} denotes which character to delete, where {id} is an integer

sample output:


        {
            "comics_appeared_in": 10563,
            "current_alias": "captain america",
            "id": 5,
            "origin": "Human enhanced to the peak of human perfection...",
            "power": "super-soldier",
            "real_name": "steven 'steve' rogers",
            "universe": "earth-616"
        }
